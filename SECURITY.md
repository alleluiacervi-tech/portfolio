# Security Policy

## Supported Versions

This repository tracks the latest `main` branch for security updates.

## Reporting a Vulnerability

Please do not create a public issue for security vulnerabilities.

Instead, report privately to the maintainers and include:

- A clear description of the issue
- Steps to reproduce
- Potential impact
- Suggested remediation if available

## Response Process

- Acknowledgement target: within 72 hours
- Initial assessment: severity and scope analysis
- Fix timeline: based on impact and exploitability
- Disclosure: coordinated release after mitigation

## Secrets and Credentials

- Never commit credentials, tokens, or private keys.
- Use environment variables and repository secrets.
